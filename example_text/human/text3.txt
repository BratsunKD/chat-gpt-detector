Роль баз данных в современных системах искусственного интеллекта.
	В современных реалиях искусственный интеллект выступает в роли одного из наиболее популярных инструментов решения разноплановых задач из разных сфер человеческой жизни. Можно сказать, что его стремительное развитие является следствием повышенного общественного интереса, агрессивного маркетинга и новых возможностей по использованию уже существующих инструментов.
	Если рассматривать системы, использующие искусственный интеллекту с точки зрения технической реализации, наибольший интерес вызывает реализация непосредственного компонента, решающего поставленную задачу, будь то нейросеть или еще какой-либо инструмент. Но нельзя не отметить низкую эффективность самого инструмента решения задачи в ваккуме: для избыточного решения требуется предоставить программе некоторое системное окружение, состоящее в том числе из входных данных, интегрированных систем, моделей хранения и модификации данных. Одним из ключевых требований к такому вспомогательному окружению является возможность оперативно взаимодействовать с большими объемами данных, которые служат для обучения нейросетей. Кроме того, в ходе работы системы могут формироваться новые данные, которые также необходимо будет в дальнейшем где-то хранить и обрабатывать, то у системы должен быть непрерывный быстрый доступ к таким данным.
	Наличие описанного выше требования предписывает разработчика системы, использующей искусственный интеллект для решения поставленных задач, организовать некоторое хранилище данных с внешних API, предоставляющим к ним доступ. При этом к самому хранилищу данных выдвигаются системные требования, касающиеся скорости его работы, надежности, отказоустойчивости и т.д. Наиболее популярной имплементацией описанной модели работы с данными выступаем база данных, В классическом примере база данных выступает как некоторая сторонняя система, которая интегрируется в финальную систему. В зависимости от типа данных и особенностей их обработки можно использовать различные базы данных. Например, существенные отличия есть между SQL и NoSQL базами данных. В случае сложной неформальной структуры данных, с которыми работает искусственный интеллект, зачастую отдается предпочтение NoSQL базам данных, например графовым или документным.
	Попробуем дать оценку необходимости интегрирования базы данных в систему, работающую с искусственным интеллектом: всегда ли наличие базы обязательно. Особенностью работы нейросетевых алгоритмов является анализ большого числа параметров и обширная классификация входных данных. Из факта наличия этой особенности можно сделать вывод, что объем входных данных для системы с искусственным интеллектом обычно больше среднего значения для абстрактной информационной системы, не использующей искусственный интеллект. Раз системе требуется работать с повышенными объемами данных, очень дорогим становится их хранение с оперативной памяти, кроме того подобное хранение не позволяет удобно и оперативно экспортировать и импортировать данные из различных форматов. Кроме того, многие форматы представления данных сами по себе не обладают удобным и, что немаловажно, быстрым API для их обработки. Например, xls или xml файлы могут содержать большое количество данных, но для удобной работы с ними придется сначала их вычитать из исходного файла, а затем переводить в кастомное представление в рабочей среде используемой системы, При этом для каждой новой модели построения данных придется определять вручную трансформеры данных модели слоя бизнес-логики, описывать эти самые модели, что усложнит и замедлит разработку и доработку самой системы за счет дополнительной рутинной работы,
	Проблема, описанная выше, может быть решена за счет использования базы данных, которая представляет унифицированный API, который удобно использовать из рабочей среды. Также большинство популярных баз данных “из коробки” или с помощью существующих библиотек позволяют экспортировать данные из популярных моделей представления. Таким образом, роль баз данных в системах искусственного интеллекта заключается в оптимизации рутинной работы по экспортированию и импортированию входных “сырых” данных, оптимальном с точки зрения ресурсов хранении этих данных и обеспечения безопасного оперативного доступа системы искусственного интеллекта (содержащей некую нейросеть/модель) к этим данным. Несомненно, некоторые простые системы могут решать поставленную задачу и не прибегая к использованию баз данных, например, если объем входных данных невелик или система уже адаптирована к работе с ними в специфическом формате, и этот формат не будет меняться в процессе работы. Также ограничением на использование баз данных в подобных системах может выступать требование на изолированность исходной системы, то есть запрет на интеграцию сторонних систем, коими по сути и являются базы данных. 
	Существуют также и некоторые классы задач, при решении которых формат поступления и формирования данных не позволяет их легко представить в базе данных, либо такое представление затрудняет функционирование системы. Например, задачи, основанные на анализе некоторых показателей в реальном времени без необходимости ретроспективы предыдущих измерений не относятся к классу задач, для решения которых требуется интегрировать в существующую систему с искусственным интеллектом базу данных.
